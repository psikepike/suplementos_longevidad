# Supplement SEO Page Audit

Use this skill when a task affects any of these routes:
- `/suplementos`
- `/suplementos/[slug]`
- `/comparativas/[keyword]`

Goal:
verify that each page is structurally correct for SEO and aligned with the affiliate editorial model.

Audit at minimum:
- canonical correctness
- metadata consistency
- single clear search intent per page
- internal linking quality
- breadcrumb coherence
- absence of legacy or duplicate routes
- absence of cannibalization between listing, product, and comparison pages
- indexability risks
- soft 404 risks

Validation evidence must include:
- exact file paths involved
- canonical strings
- metadata outputs when applicable
- explicit search results for legacy paths or duplicate route patterns
- concrete examples of internal links

Do not:
- close SEO tasks without checking canonical and internal links
- use abstract wording such as "SEO ok" or "estructura limpia"
- assume runtime metadata is correct without verification when build/runtime validation is required
